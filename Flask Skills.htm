Flask Skills

Flask Skills is meant to help you practice and assess your understanding of the Flask framework.

Follow the directions below to review what you should have learned, practice implementing Flask, and submit an assessment of your understanding.

Take Aways

  • know how to construct a basic Flask application with multiple routes
  • know when to use GET vs. POST methods
  • know how to retrieve form input
  • know how to build and integrate Jinja templates and extend base templates
  • know how to create and run a new virtual environment

Assessment

Be honest with yourself and your adviser about how this assessment goes for you. If you don’t finish it, take note of which problems you weren’t able to finish, and come back to them later. If you have ideas about how to solve something, but not a fully formed solution, leave your thoughts as comments in the function body.

We’re assessing this on correctness and style — we want to see where you are on all of these things. So, choose good variable names, and show off good Python style. In addition, for each question, consider any “edge cases” where you might want to handle unusual input or errors and, as you are able to, think about how to handle those. If it makes sense to use a more intermediate concept, like a list comprehension, do so.

You must turn in all of your work through FRODO by 9PM on Sunday.

Directions

Setup

First, create a virtual environment and activate it.

Reminder: in Windows, make sure you use the command virtualenv env --always-copy to create a new virtual environment.

We’ve included the requirements for this project in requirements.txt. Install them.

Part 1: Flask Application

Write a small Flask app in server.py that does the following:

  • Serves the template index.html at the route /. Feel free to be as creative as you’d like with the homepage. Do, however, make sure to include a link to the application form.

  • Serves the template application-form.html at the route /application-form.

    Make a copy of both the application-form.html and styles.css files you made in skills-html-css. Modify application-form.html to create your job-selection input using a jinja for-loop. (Hint: this means you’ll have to pass the list of possible jobs from your route to the template.)

  • Handles submission of a form in application-form.html to the route /application-success.

    • Gets the first name, last name, salary, and job title from the form submission.

    • Returns a response that acknowledges their application. This should repeat their name, title, and salary requirement, like:

      Thank you, Jessica McHackbright, for applying to be a QA Engineer. Your
minimum salary requirement is $89,000.

    You should do this by using a Jinja template. Use the file called application-response.html.

    You’ll need to look up currency formatting to learn how to make the salary appear with commas in the right places, and, if needed, a decimal point and exactly two decimal places.

Part 2: Template Inheritance

Create a new template called base.html. Have your other three templates inherit from base.html using Jinja’s {% extends %} and {% block %} syntax.
